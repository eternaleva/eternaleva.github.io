简介

配置

依赖

pojo

mysql

mysql和pojo的属性对应，pojo驼峰命名在mysql中以下划线_分割大小写

mapper

创建对应mapper接口，继承BaseMapper（注意指定范型）

日志输出

properties或者yml中:
mybatis-plus.configuration.log-impl: xxx

操作

插入

insert(POJO) BaseMapper
save(POJO) IService

更新

updateById(POJO)

删除

单个删除

deleteById(Id)

批量删除

deleteBatchIds(Arrays.asList(1,2,3))

Map条件删除

deleteByMap(map)

逻辑删除

表中添加deleted字段，POJO属性中同样，并使用注解@TableLogic
0为存在，1为删除
最好配置上自动填充。
deletedById(Id)

查询

动态批量查询

比如按照id为1,2,3来批量查询：userMapper.selectBatchIds(Arrays.asList(1,2,3))

Map条件查询

map.put(“name”, “tian”);
userMapper.selectByMap(map);

分页查询

配置类中注册分页插件

使用插件中的Page对象来获取（参数：当前页，每页记录数）

  1. selectPage(page, null);
  2. selectMapsPage(page, null)

wrapper

条件构造器

大小比较

  1. lt
  2. gt
  3. le
  4. ge

相等比较

  1. eq
  2. ne

空比较

  1. isNull
  2. isNotNull

区间比较

  1. between
  2. notBetween

排序

  1. orderBy
  2. orderByDesc
  3. orderByAsc

模糊查询

  1. like
  2. notLike
  3. likeLeft
  4. likeRight

策略

主键策略（雪花算法）

POJO的属性上
@TableId(type=IdType.ASSIGN_ID)

  1. ASSIGN 生成19位id
  2. 生成UUID
  3. AUTO 自增长
  4. INPUT 自己放入

雪花算法：分布式ID算法
0 符号位
1-42 时间戳
43-52 工作进程
53-64 序列号

全局主键配置

另外可以设置全局主键配置

自动填充

  1. 在Pojo中的属性使用@TableFiled(xxx, fill = ….)注解
  2. 创建类实现MetaObjectHandler接口，重写insert和update操作
  3. 用@component注册到组建里
    在执行相应操作的时候，会自动填充对应内容

    乐观锁

    简介

    防止并发操作，出现丢失更新情况。
    采用版本号，对照版本号执行操作
  4. 版本号一样，提交，版本号改变
  5. 版本号不同，回滚

如何保证版本号是不会出更新问题？
使用volatile修饰，保证一致性。

使用乐观锁

  1. 表字段中添加version字段，并在pojo中添加对应属性，使用@version注解
  2. 创建配置类MybatisPlusConfig，启用事务注解（@EnableTransactionManagement），启用mapper扫描（@MapperScan），此时可以删除启动类的MapperScan
  3. 注册乐观锁MybatisPlusInterceptor

其他

mapper的xml如果在resources下

要在配置文件（yml）中添加mapper-locations
