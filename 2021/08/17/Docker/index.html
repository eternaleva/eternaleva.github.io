为什么会出现Docker

主要是当前技术存在问题，所以需要新的东西来去解决。

Docker历史

dotCloud — 做pass的云计算服务，慢慢做了一套linux容器化技术
然后将该技术命名为Docker。
2013年，公司活不下去，然后宣布开源。
使用人越来越多后，每个月更新一个版本。
Docker技术之前，比较火的是虚拟机技术，Docker是容器技术，本质其实也是虚拟机技术。
Docker基于Go语言开发

官网

https://www.docker.com
文档地址：https://docs.docker.com
仓库地址： https://hub.docker.com

容器化技术和虚拟机技术的区别

虚拟机技术：虚拟一个完整的系统，包含各种硬件，安装和运行应用
容器化技术：模拟的不是全部的系统，容器直接运行在宿主机的内核，容器没有自己的内核，也没有虚拟硬件，且容器间是隔离的，互不干扰。

作用

应用更快速的开发和部署

更好的升级和扩缩容

更简单的系统运维

更高效的资源利用

Docker组成

镜像（image）

镜像有点像一个模板，容器可以按照这个模板执行操作。

容器（container）

容器，运行的镜像，独立运行一个或一组应用，一个简易的环境。
启动、停止、删除等基本命令。

仓库（repository）

存放镜像的地方
私有仓库、公有仓库。

docker version

镜像命令

docker images [可选参数]

查看本地镜像
Repository：镜像仓库源
TAG：镜像标签
IMAGE ID：镜像ID
CREATED：镜像创建时间
SIZE：镜像大小

可选参数：
-a：-all
-q：-quiet 只显示id

docker search [可选参数]

docker仓库搜索

可选参数：
–filter=条件

docker pull [镜像名:tag]

下载镜像，如果不写tag，默认是下载latest
docker是分层下载，已经拥有的东西不会下载。

docker rmi [id…]

  1. docker rmi id
    id可以简写，部分字符即可以确定唯一id镜像
  2. docker rmi id1 id2 id3
    删除多个镜像
  3. docker rmi -f $(docker images -aq)
    全部删除镜像（递归删除）

容器命令

docker run [可选参数] image

可选参数：
–name=”name”：指定容器名字l
-d：后台方式运行
-it：-i -t的缩写，使用交互方式运行，进入容器查看内容
-P：指定端口 主机映射端口：容器端口
-p：随机端口

docker start [容器id]

docker restart [容器id]

docker stop [容器id]

docker kill [容器id]

docker ps [可选参数]

可选参数：
-a：列出当前正在运行的容器 + 历史运行的容器（所有）
-n=x：显示最近创建的x个容器
-aq：当前所有运行的容器

退出容器

exit：停止并退出容器
ctrl + p + q：退出容器（不停止）

docker rm [可选参数] [容器id]

删除容器（不能删除正在运行的）
强制删除 docker rm -f $(docker ps -aq)

常用其他命令

docker run -d [容器名]

后台启动

docker容器使用后台运行的时候，就必须要有前台进程。而当docker发现没有前台应用的时候，就会自动停止。

docker logs -f -n num -t [容器id]

查看日志
-f：Follow log output
-t：Show timestamps
-n num：Number of lines to show from the end of the logs

docker top [容器id]

查看容器内的信息

docker inspect [容器id]

查看镜像的元数据

docker exec -i -t [容器id] /bin/bash

进入运行的容器，并启动一个新的终端，执行命令。
我们有时候需要进入容器，修改一些配置

docker attach [容器id]

attach 不会在容器中创建进程（终端）执行额外的命令，只是附着到容器上.

docker cp 容器id:容器内路径 主机路径

复制容器的内容到主机。

拷贝是手动，后续可以使用-v 来实现自动管理

docker stats

查看docker运行状态，没有u

Docker镜像原理

docker镜像是一个独立的软件包，包括代码、运行时、环境库以及变量和配置文件等。

镜像加载原理

UnionFS（联合文件系统），分层、轻量级并且高性能的文件系统。
支持对文件系统的修改作为一次一次的提交叠加。可以把不同目录的文件挂载到同一个虚拟文件系统下。

BootFS

包含bootloader和kernel，容器启动时，会加载BootFS的系统文件。当boot加载完成，容器系统的内核代码均在内存中，然后将内存使用权由bootfs交给内核，此时容器会卸载BootFS

RootFS

标准linux的目录和文件系统
