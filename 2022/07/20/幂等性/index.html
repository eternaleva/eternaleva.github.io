幂等性

发表于 2022-07-20 更新于 2024-04-21 分类于 解决方案
本文字数： 392 阅读时长 ≈ 1 分钟

简介

应用场景

本质是针对重复：

  1. 可能引起的重复请求（网络、用户重复操作）
  2. 应用重试机制
  3. 第三方接口的异步回调
  4. 中间件导致的重复请求问题
  5. 定时任务
  6. 等等….

解决方案

前端控制

  1. 按钮置灰
  2. token机制
  3. PRG模式（Post-Redirect-Get）
  4. Session中存储特殊标志

后端控制

根据业务选择合适的方案

  1. 唯一索引
  2. Token + Redis
  3. 状态机幂等
  4. 乐观锁实现
  5. 防重表
  6. select + insect
  7. 分布式锁
  8. 缓冲队列
  9. 全局唯一实现幂等

Token + Redis的实现

适用于

增 删 改 幂等性场景

业务流程

  1. 申请Token（生成Token，并放到Redis）
  2. 取Token
  3. 做支付（验证Token，支付成功则删除Token）

缺点：需要跟系统做两次交互，性能有所不佳

代理逻辑

AOP来实现

  1. AOP生成Token，并放入Redis
  2. tokenService：对特定的API方法获取Redis里的Token值
  3. AOP实现Token验证及删除（保证原子性）
